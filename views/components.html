Widget

MortarJs Widget is a core component that integrates with the jQuery plugin ecosystem and synchronizes operations such as loading dynamic html fragments, data models, and style sheets.

How do you use Widget?

1. Define your Widget

In this step, you define default options, callback when the widget instance is ready for business, and a destructor. These are generally the only bits you will ever need when defining a Widget. But since the underlying engine is jQuery UI widget factory, you can specify other properties/methods that jQuery UI widget factory can use. Please see jQuery UI widget factory for more details.
widget("mortar.main", { // Default options options: { }, // Callback when widget is ready for business _create: function() { }, // Destructor _destroy: function() { } });

FYI

1. MortarJs Widget provides the same core interfaces as jQuery UI widget factory to enable a migration path from existing jQuery systems.

2. Widget names must conform to the namespace.name requirement imposed by jQuery UI widget factory. So, a valid widget name is "mortar.main", but not "main" or "mortar".

2. Instantiate your Widget

Since MortarJs Widgets are jQuery plugins, you would just use them as such.
$("body").main();

3. Give meaning to your Widget!

An important part of modern Web Appplication is loading data from a server. And in more advanced scenarios, loading dynamic html template for rendering your data. MortarJs Widget makes this easy for you by loading html fragments, server data, and synchronizing these operations so that you can safely add your behavior to the widget instance.
widget("mortar.main", { // Default options options: { template: { "url": "http://template.html" }, model: { "url": "http://data.json" } }, // Callback when widget is ready for business _create: function() { console.log("Your html template and server data are loaded at this point"); }, // Destructor _destroy: function() { } });

FYI

1. A common issue for KnockoutJs developers is knowing when to call ko.applyBindings and that it can be done only once. MortarJs Widget takes care of this for you.

View

construct from core widget, which automatically load html fragments and style sheets. Difference between this and widget is that widget a path for html fragments and css files need to be specified in order to get loaded. View will automatically load these files from the default folder "views", unless explicitly disabled when instantiating your views.

Model

derived from widget to provides a transparent system for dynamic loading of data from a url or a locally.

Style

derived from widget to provide a transparent system for loading style sheets dynamically from a url or locally.

Fragment

derived from widget to provide a transparent system for loading html fragments dynamically from a url or locally.

Ko Factory

provides a simple interface to create KnockouJs observables. It is designed for simplicity, efficiency, and speed.
